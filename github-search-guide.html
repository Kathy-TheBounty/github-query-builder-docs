1. Fonctionnement général

Mots-clés vs Qualifiers

Mots-clés libres Qualifiers qualifier:valeur
Cherchent dans plusieurs champs simultanément Ciblent un champ précis uniquement
Supportent AND, OR, NOT explicites OR via répétition, NOT via -qualifier:val
"machine learning" → expression exacte language:python → champ précis
💡 Règle d'or : AND/OR/NOT explicites ne fonctionnent qu'entre mots-clés libres. Pour les qualifiers, tout est implicite — espace = AND, répétition = OR, tiret = NOT.

2. Recherche Users

Ce que cherchent les mots-clés

Un mot-clé sans qualifier cherche simultanément dans : username, nom affiché, email public (partie locale seulement), bio du profil.

💡 La bio est un champ intéressant pour le sourcing. Les devs y mettent leur stack, leur titre, leurs spécialités.

Qualifiers disponibles

QualifierChamp cibléExempleNotes
language: Langage principal des repos publics language:python OR implicite si répété
location: Champ location du profil (déclaratif) location:Paris OR implicite si répété
followers: Nombre de followers followers:>50 followers:20..200 Supporte ranges
repos: Nombre de repos publics repos:>10 repos:5..50 Inclut les forks
created: Date de création du compte created:>2019-01-01 Format ISO YYYY-MM-DD
in:login Username (recherche partielle) python in:login Mot-clé + in:
in:name Nom affiché Sophie in:name Mot-clé + in:
in:email Email public (partie locale) dataiku in:email Pas le domaine
type: Type de compte type:user Exclut les orgs

Location — le grand angle mort

Le champ location: est entièrement déclaratif. Un dev parisien peut écrire "Paris", "75", "Île-de-France", "Greater Paris Area" ou rien du tout.

⚠️ La seule façon de couvrir une zone exhaustivement : lancer plusieurs queries en parallèle avec toutes les variantes — et agréger les résultats.
type:user language:python location:Paris location:"Île-de-France" location:France location:75 followers:>20

Exemples de requêtes Users

"ML engineer" language:python location:Paris location:France followers:>20 repos:>5 NOT "junior" "tech lead" OR "engineering manager" location:France language:typescript repos:>5 "open to work" "python" location:Paris language:python

3. Recherche Repos

Ce que cherchent les mots-clés par défaut

Sans in:, un mot-clé cherche dans nom + description + topics. Pas dans le README.

Stratégie Repos → Users

La vraie puissance : trouver des repos pertinents → extraire leurs owners et contributeurs → enrichir leurs profils. Cela contourne la limite de la recherche users qui ne peut pas filtrer par framework ou librairie spécifique.

4. Logique booléenne

OpérateurSyntaxeFonctionne surExemple
AND implicite Espace entre qualifiers différents Tous les qualifiers language:python location:France
OR implicite Répéter le même qualifier Tous les qualifiers location:Paris location:Lyon
OR explicite OR Mots-clés libres uniquement "frontend" OR "front-end"
NOT explicite NOT Mots-clés libres uniquement "developer" NOT "junior"
NOT (tiret) -qualifier:valeur Tous les qualifiers -language:php

Exemples de combinaisons

// AND implicite entre qualifiers différents language:python location:France repos:>10 followers:>50 // OR implicite — même qualifier répété location:Paris location:Lyon location:Bordeaux location:Nantes // Combiner keywords et qualifiers "machine learning engineer" location:Paris location:"Île-de-France" language:python followers:>20 // Exclusions location:France -language:php -language:java NOT "junior" NOT "intern"
⚠️ Interdit : OR entre qualifiers différents (location:Paris OR language:python) et AND entre mêmes qualifiers (language:python AND language:js) ne fonctionnent pas. Contournement : deux queries séparées.

5. Limites & contournements

Dépasser les 1 000 résultats — pagination temporelle

Quand une query retourne exactement 1 000 résultats, il en existe plus. Découper sur created: :

"python developer" location:France created:<2018-01-01 "python developer" location:France created:2018-01-01..2020-12-31 "python developer" location:France created:2021-01-01..2022-12-31 "python developer" location:France created:>2023-01-01

AND entre langages — technique d'intersection

Impossible en une query. Simuler via intersection :

Query 1 : language:python location:France → liste_A Query 2 : language:typescript location:France → liste_B Intersection A ∩ B → profils présents dans les deux listes
⚠️ Cette technique donne une approximation — un dev polyvalent peut n'apparaître que dans une liste si ses repos dans l'autre langage sont minoritaires.

Exclure les langages parasites

machine learning language:python -language:"jupyter notebook" -language:html stars:>20

Interface web vs API REST

Interface webAPI REST
Résultats par page30100 avec per_page=100
Rate limitLimité par session30 req/min (sans token), 5 000/h (avec)
ParallélismeNonOui — plusieurs tokens
Post-filtrageNonOui — résultats JSON filtrables

6. Cheat sheet — Référence complète

Mots-clés vs Qualifiers — la différence fondamentale

Mots-clés libres Qualifiers qualifier:valeur
Ce que c'est Termes tapés sans syntaxe spéciale, avec ou sans guillemets Syntaxe qualifier:valeur ciblant un champ précis
Où ça cherche — Users Username · nom affiché · email public (partie locale) · bio Un champ précis seulement (ex. language: → langage principal)
Où ça cherche — Repos Nom · description · topics (par défaut) — modifiable avec in: Un attribut précis seulement (ex. stars: → nombre d'étoiles)
AND Juxtaposer : "backend" "python" Juxtaposer des qualifiers différents : language:python location:France
OR Mot-clé OR mot-clé : "frontend" OR "front-end" (max 5 opérateurs) Répéter le même qualifier : location:Paris location:Lyon
NOT / Exclure Mot-clé NOT mot-clé : "developer" NOT "junior" Préfixer avec - : -language:php
⛔ Ce qui ne marche pas OR/NOT ne fonctionnent pas sur les qualifiers OR/AND explicites ne fonctionnent pas entre qualifiers

Logique — mots-clés (Users & Repos)

SyntaxeComportementExemple
mot Mot seul — cherche le terme dans les champs par défaut developer
"expression exacte" Les mots doivent apparaître dans cet ordre exact "machine learning engineer"
"mot1" "mot2" AND implicite — les deux expressions doivent être présentes "backend" "python"
"A" OR "B" OR explicite — l'un ou l'autre suffit. Max 5 opérateurs OR/NOT par query. "frontend" OR "front-end" OR "ui developer"
"A" NOT "B" NOT explicite — exclut B de tous les champs cherchés "developer" NOT "junior" NOT "intern"
mot in:champ Restreint la recherche à un champ précis (users : login/name/email — repos : name/description/topics/readme) python in:login · LangChain in:readme
"A" OR "B" in:champ OR entre mots-clés + ciblage dans un champ précis "llm" OR "rag" in:name,description
⚠️ Rappel : OR et NOT explicites ne fonctionnent qu'entre mots-clés. Ils ne s'appliquent pas aux qualifiers.
